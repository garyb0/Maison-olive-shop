🖥️ 2. Choix de l'hébergement

💡 Recommandation : VPS avec Ubuntu 22.04 LTS

Pourquoi un VPS ?

  • Contrôle total sur l'environnement
  • Meilleures performances que l'hébergement mutualisé
  • Prix raisonnable (5-10€/mois)
  • Support de PM2 pour garder l'app active

Options populaires :

Fournisseur Prix/mois RAM Stockage Idéal pour
Hetzner ~4€ 4GB 40GB Meilleur rapport qualité/prix
DigitalOcean ~6€ 1GB 25GB Interface simple
Vultr ~5€ 1GB 25GB Bonne performance
OVH ~5€ 2GB 40GB Serveurs en France
🎯 Mon choix recommandé : Hetzner CX21 (4€/mois, 4GB RAM, 40GB SSD)

🔧 4. Configuration DNS

Étape 1 : Obtenez l'IP de votre serveur

Une fois votre VPS créé, vous recevrez une IP publique :

Exemple : 123.456.789.012

Étape 2 : Configurez les enregistrements DNS

Dans le panneau DNS de votre registrar, ajoutez :

Enregistrements obligatoires :

Type Nom Valeur TTL
A @ IP_DE_VOTRE_SERVEUR 3600
A www IP_DE_VOTRE_SERVEUR 3600
AAAA @ IPv6_SI_DISPONIBLE 3600

Étape 3 : Attendez la propagation

  • Délai : 5 minutes à 48 heures
  • Vérifiez : avec nslookup votredomaine.com ou whatsmydns.net

⚙️ 5. Installation sur le serveur

Étape 1 : Connectez-vous en SSH

ssh root@IP_DE_VOTRE_SERVEUR

Étape 2 : Mise à jour du système

apt update && apt upgrade -y

Étape 3 : Installez Node.js

# Installez Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Vérifiez l'installation
node --version
npm --version

Étape 4 : Installez PM2

npm install -g pm2

Étape 5 : Installez Nginx

apt install -y nginx
systemctl enable nginx
systemctl start nginx

Étape 6 : Clonez votre projet

# Créez un utilisateur dédié (sécurité)
adduser chezolive
usermod -aG sudo chezolive
su - chezolive

# Clonez le repository
cd /home/chezolive
git clone https://github.com/votre-username/Chez-olive-shop.git
cd Chez-olive-shop

Étape 7 : Installez les dépendances

npm ci

Étape 8 : Générez le client Prisma

npx prisma generate

🔒 6. Configuration SSL/HTTPS

⚠️ Important : Ne jamais utiliser un certificat auto-signé en production !

Option A : Let's Encrypt (Gratuit et recommandé)

# Installez Certbot
apt install -y certbot python3-certbot-nginx

# Obtenez un certificat
certbot --nginx -d votredomaine.com -d www.votredomaine.com

# Configurez le renouvellement automatique
crontab -e
# Ajoutez cette ligne :
0 12 * * * /usr/bin/certbot renew --quiet

Option B : Certificat auto-signé (pour les tests uniquement)

openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/nginx-selfsigned.key \
  -out /etc/ssl/certs/nginx-selfsigned.crt

🔐 7. Configuration des variables d'environnement

Étape 1 : Créez le fichier .env

cp .env.example .env
nano .env

Étape 2 : Configurez les variables

# Base de données (Turso - gratuit)
DATABASE_URL="libsql://votre-base.turso.io"
DATABASE_AUTH_TOKEN="votre-token-turso"

# Sécurité session (GÉNÉREZ UN VRAI SECRET !)
SESSION_SECRET="$(openssl rand -hex 32)"
SESSION_COOKIE_NAME="chezolive_session"
SESSION_DURATION_DAYS="30"

# Business rules
TAX_RATE="0.14975"
SHIPPING_FLAT_CENTS="899"
SHIPPING_FREE_THRESHOLD_CENTS="7500"

# URLs (IMPORTANT : HTTPS !)
NEXT_PUBLIC_SITE_URL="https://votredomaine.com"

# Stripe (optionnel)
STRIPE_SECRET_KEY="sk_live_..."
STRIPE_WEBHOOK_SECRET="whsec_..."

# Support / business (UTILISEZ UN EMAIL PROFESSIONNEL !)
BUSINESS_SUPPORT_EMAIL="support@votredomaine.com"

# Email transactionnel (Resend)
RESEND_API_KEY="re_..."
RESEND_FROM_EMAIL="Chez Olive "

Étape 3 : Validez la configuration

npm run validate:env:prod

🚀 8. Déploiement avec PM2

Étape 1 : Créez le fichier ecosystem.config.cjs

module.exports = {
  apps: [{
    name: 'chez-olive-shop',
    script: 'node_modules/.bin/next',
    args: 'start',
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '1G',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
};

Étape 2 : Build et démarrage

# Build de production
npm run prisma:generate
npm run build

# Démarrez avec PM2
pm2 start ecosystem.config.cjs

# Sauvegardez la config PM2
pm2 save

# Configurez le démarrage automatique
pm2 startup

Étape 3 : Vérifiez le statut

pm2 status
pm2 logs chez-olive-shop

🌐 9. Configuration Nginx (reverse proxy)

Étape 1 : Créez la configuration Nginx

sudo nano /etc/nginx/sites-available/Chez-olive-shop

Étape 2 : Ajoutez la configuration

server {
    listen 80;
    listen [::]:80;
    server_name votredomaine.com www.votredomaine.com;

    # Redirection HTTPS (optionnel mais recommandé)
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name votredomaine.com www.votredomaine.com;

    # Certificats SSL (Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/votredomaine.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/votredomaine.com/privkey.pem;

    # Configuration SSL sécurisée
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
    ssl_prefer_server_ciphers off;

    # Headers de sécurité
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";

    # Proxy vers Next.js
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # Fichiers statiques (optimisé)
    location /_next/static {
        proxy_pass http://127.0.0.1:3000;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    # Images (optimisé)
    location ~* \.(jpg|jpeg|png|gif|ico|svg)$ {
        proxy_pass http://127.0.0.1:3000;
        expires 30d;
        add_header Cache-Control "public";
    }
}

Étape 3 : Activez le site

# Créez le lien symbolique
sudo ln -s /etc/nginx/sites-available/Chez-olive-shop /etc/nginx/sites-enabled/

# Testez la configuration
sudo nginx -t

# Redémarrez Nginx
sudo systemctl restart nginx

🔧 11. Maintenance et monitoring

Commandes PM2 utiles

# Voir les logs en temps réel
pm2 logs chez-olive-shop

# Redémarrer l'application
pm2 restart chez-olive-shop

# Voir les métriques
pm2 monit

# Sauvegarder la base de données
npm run db:backup

Sauvegardes automatiques

Créez un script de backup :

nano ~/backup-Chez-olive.sh
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/home/chezolive/backups"

# Backup de la base de données
cd /home/chezolive/Chez-olive-shop
npm run db:backup

# Backup du code
tar -czf $BACKUP_DIR/Chez-olive-code-$DATE.tar.gz \
  --exclude=node_modules \
  --exclude=.next \
  --exclude=.git \
  .

# Supprimez les backups de plus de 30 jours
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

Configurez le crontab pour une sauvegarde quotidienne :

# Rendez le script exécutable
chmod +x ~/backup-Chez-olive.sh

# Ajoutez au crontab
crontab -e
# Ajoutez :
0 3 * * * /home/chezolive/backup-Chez-olive.sh

Monitoring avec PM2 Plus (optionnel)

# Connectez-vous à PM2 Plus
pm2 plus

# Accédez au dashboard en ligne
# https://app.pm2.io/

Logs Nginx

# Voir les logs d'accès
sudo tail -f /var/log/nginx/access.log

# Voir les logs d'erreur
sudo tail -f /var/log/nginx/error.log

🔧 12. Dépannage courant

Problème : Le site ne se charge pas

Vérifications : 
# 1. PM2 tourne-t-il ?
pm2 status

# 2. Nginx tourne-t-il ?
sudo systemctl status nginx

# 3. Le port 3000 est-il ouvert ?
netstat -tlnp | grep 3000

# 4. Les logs PM2
pm2 logs chez-olive-shop --lines 50

# 5. Les logs Nginx
sudo tail -f /var/log/nginx/error.log

Problème : Erreur 502 Bad Gateway

Causes possibles :
  • Next.js n'est pas démarré
  • Le port est incorrect
  • Problème de permissions
Solution : 
# Vérifiez que Next.js écoute sur le bon port
pm2 logs chez-olive-shop

# Redémarrez PM2
pm2 restart chez-olive-shop

Problème : Les images ne se chargent pas

Vérifications : 
# Vérifiez les permissions des fichiers
ls -la /home/chezolive/Chez-olive-shop/public/

# Corrigez les permissions si nécessaire
sudo chown -R chezolive:chezolive /home/chezolive/Chez-olive-shop

Problème : Les emails ne s'envoient pas

Vérifications : 
# 1. Vérifiez la variable RESEND_API_KEY
grep RESEND_API_KEY /home/chezolive/Chez-olive-shop/.env

# 2. Testez l'envoi d'email
curl -X POST 'https://api.resend.com/emails' \
  -H 'Authorization: Bearer re_...' \
  -H 'Content-Type: application/json' \
  -d '{"from":"test@resend.dev","to":"votre@email.com","subject":"Test","html":"
Test
"}'

Problème : Erreur de base de données

Vérifications : 
# 1. Vérifiez la connexion à Turso
npx prisma db push

# 2. Vérifiez les migrations
npx prisma migrate status

# 3. Appliquez les migrations si nécessaire
npx prisma migrate deploy
